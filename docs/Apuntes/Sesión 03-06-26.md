## 04 Tracking de objetos

El tracking de objetos permite asignar un identificador único (ID) a cada objeto detectado para poder seguirlo a lo largo de los diferentes frames de un video.

YOLO se encarga de detectar los objetos en cada frame, mientras que el tracker intenta mantener el mismo ID para un objeto aunque este se encuentre en posiciones distintas conforme avanza el video.

En Supervision existen dos formas de utilizar ByteTrack. La primera es mediante ByteTrack incluida directamente en Supervision. La segunda es mediante el paquete trackers utilizando ByteTrackTracker, que corresponde a la implementación más actual.

El método update_with_detections permite actualizar las detecciones utilizando ByteTrack, mientras que en ByteTrackTracker se utiliza el método update.

Los IDs generados por el tracker pueden mostrarse mediante etiquetas utilizando LabelAnnotator, permitiendo identificar visualmente cada objeto rastreado.

TraceAnnotator permite dibujar una trayectoria utilizando el centroide de cada objeto. Conforme un objeto se mueve, se genera una línea que muestra el recorrido realizado durante el seguimiento.

También es posible almacenar información adicional asociada a cada ID, por ejemplo, contar cuántos frames lleva visible un objeto mediante un diccionario que se actualiza durante el procesamiento del video.

El proceso general consiste en detectar objetos, actualizar el tracker, generar etiquetas utilizando los IDs y finalmente anotar los resultados sobre cada frame antes de guardar o visualizar el video.

### Conclusión

En esta práctica aprendí a realizar seguimiento de objetos utilizando ByteTrack y Supervision. Comprendí cómo asignar y mantener IDs únicos para cada objeto detectado, cómo visualizar su trayectoria mediante TraceAnnotator y cómo mostrar información adicional asociada a cada ID. También entendí la diferencia entre ByteTrack y ByteTrackTracker, así como la forma de integrarlos dentro del flujo de procesamiento de video.

## 05 Zonas y conteo

El objetivo de una PolygonZone es detectar si un objeto se encuentra dentro de un área determinada definida mediante un polígono.

Los polígonos se crean a partir de una lista de puntos, por lo que al modificar sus coordenadas es posible cambiar completamente la zona de interés.

El método trigger() permite verificar qué detecciones se encuentran dentro de la zona y actualiza automáticamente el contador current_count.

PolygonZone es útil para responder preguntas como cuántos objetos hay dentro de una región específica en un momento determinado.

También existen las LineZone, que utilizan una línea en lugar de un área para detectar cuándo un objeto la cruza.

A diferencia de PolygonZone, LineZone no mide presencia sino flujo de objetos. Su función es contar cuántos objetos cruzan una línea en una dirección u otra.

El método trigger() actualiza automáticamente los contadores in_count y out_count cuando un objeto atraviesa la línea.

Para que el conteo funcione correctamente es necesario utilizar tracking, ya que LineZone necesita identificar cada objeto mediante su tracker_id.

PolygonZoneAnnotator permite visualizar el polígono y el número actual de objetos dentro de la zona, mientras que LineZoneAnnotator muestra la línea y los cruces acumulados.

También es posible utilizar PolygonZone y LineZone al mismo tiempo dentro de un mismo video para analizar tanto la ocupación de una región como el flujo de objetos que la atraviesan.

### Conclusión

En esta práctica aprendí a utilizar PolygonZone y LineZone para realizar análisis espaciales sobre las detecciones generadas por YOLO. Comprendí cómo definir áreas mediante polígonos, contar objetos presentes dentro de una región y registrar cruces a través de una línea utilizando tracking. También entendí la diferencia entre medir ocupación mediante PolygonZone y medir flujo mediante LineZone, así como la forma de combinar ambas herramientas dentro de un mismo procesamiento de video.
