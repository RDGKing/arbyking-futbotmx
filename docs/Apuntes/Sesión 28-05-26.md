## 01 Introducción a supervision
YOLO (You Only Look Once) es un modelo de visión por computadora que nos ayuda a detectar los objetos que aparecen en una imagen o video.

La librería OpenCV (cv2) nos permite trabajar con imágenes y realizar tareas como dibujar cuadros, textos o modificar imágenes desde Python.

NumPy se utiliza para trabajar con arreglos de datos y realizar operaciones numéricas de manera eficiente.

Matplotlib ayuda a visualizar y mostrar imágenes o gráficas de los resultados obtenidos.

Pathlib permite trabajar con rutas y directorios de forma más sencilla, por ejemplo para crear carpetas o acceder a archivos.

Un Bounding Box es el rectángulo más pequeño que puede envolver un objeto detectado. Este se representa mediante las coordenadas x1, y1, x2 y y2.

Cuando YOLO detecta un objeto, devuelve información como las coordenadas del Bounding Box, el identificador de clase (class_id), la confianza de la detección (confidence) y otros datos relacionados.

El valor de confidence indica qué tan seguro está el modelo de que el objeto detectado pertenece a una determinada categoría.

YOLO cuenta con diferentes versiones y tamaños de modelos, como Nano, Small, Medium, Large y Extra Large, cada uno con diferentes niveles de precisión y velocidad.

Para el reconocimiento de objetos, YOLO utiliza comúnmente el dataset COCO, el cual contiene 80 categorías diferentes de objetos.

### Conclusión

En esta práctica aprendí los conceptos básicos de la librería Supervision y cómo utilizar YOLO para detectar objetos dentro de una imagen. También comprendí qué es un Bounding Box, qué información devuelve una detección y cómo interpretar el valor de confianza que proporciona el modelo. Además, aprendí a visualizar los resultados y a guardar la información obtenida en formato JSON para poder utilizarla posteriormente.

## 02 Anotación y visualización

Los Annotators son herramientas de la librería Supervision que permiten agregar elementos visuales sobre las detecciones realizadas por un modelo de visión por computadora.

Una anotación es cualquier elemento gráfico que ayuda a representar información de una detección, por ejemplo cuadros, etiquetas, puntos o resaltados.

- El BoxAnnotator se utiliza para dibujar Bounding Boxes alrededor de los objetos detectados.

- El RoundBoxAnnotator funciona de manera similar al BoxAnnotator, pero dibuja cuadros con esquinas redondeadas.

- El LabelAnnotator permite mostrar información adicional sobre cada detección, como el nombre de la clase detectada y el valor de confianza.

- El HaloAnnotator agrega un efecto de resaltado alrededor de los objetos para hacerlos más visibles dentro de la imagen.

- El EllipseAnnotator dibuja una elipse alrededor de los objetos detectados en lugar de utilizar un rectángulo.

- El DotAnnotator coloca un punto en el centro de cada objeto detectado, lo que ayuda a identificar su posición exacta.

- El TriangleAnnotator utiliza triángulos para señalar la ubicación de las detecciones dentro de la imagen.

Los Annotators pueden personalizarse modificando propiedades como el color, el grosor de las líneas, el tamaño del texto o el tamaño de los puntos.

La librería Supervision proporciona colores predefinidos mediante la clase Color, por ejemplo Color.RED, Color.GREEN y Color.BLUE.

El orden en que se aplican los Annotators es importante, ya que las anotaciones agregadas al final se dibujan sobre las anteriores.

### Conclusión

En esta práctica aprendí a utilizar los diferentes Annotators disponibles en la librería Supervision para visualizar las detecciones realizadas por un modelo. También comprendí cómo personalizar colores, tamaños y estilos de las anotaciones, además de la importancia del orden en que se aplican. Finalmente, aprendí a combinar varios Annotators en una misma imagen para mejorar la visualización y comprensión de los resultados obtenidos.

## 03 Filtrado y Manipulación de detecciones

Las detecciones generadas por YOLO pueden filtrarse utilizando diferentes condiciones dentro de la estructura sv.Detections.

El valor de confianza (confidence) permite conservar únicamente las detecciones que superan un umbral determinado.

También es posible filtrar objetos por clase utilizando el atributo class_id.

Las máscaras booleanas permiten seleccionar únicamente las detecciones que cumplen una condición específica.

Se pueden combinar varias condiciones mediante operadores lógicos para realizar filtros más precisos.

La técnica Non-Maximum Suppression (NMS) ayuda a eliminar detecciones duplicadas que corresponden al mismo objeto.

Las detecciones también pueden filtrarse según el tamaño de su Bounding Box utilizando el atributo area.

### Conclusión
En esta práctica aprendí a filtrar detecciones utilizando criterios como la confianza, la clase y el tamaño de los objetos. También comprendí cómo eliminar detecciones duplicadas mediante NMS y cómo seleccionar únicamente los objetos que me interesan para realizar análisis más específicos.
